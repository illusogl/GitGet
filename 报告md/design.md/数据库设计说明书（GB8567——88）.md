# 数据库设计说明书

## 1 引言

### 1.1 编写目的

本数据库设计说明书旨在对"GitGet——GitHub Release 应用商店系统"的本地数据库进行详细设计说明，包括数据库的逻辑结构、物理结构、数据字典和存取方式，为后端开发人员提供数据持久化层的完整参考。预期的读者包括系统架构师、后端开发工程师、测试工程师及数据库管理人员。

### 1.2 背景

- **a. 待开发软件系统的名称**：GitGet（GitHub Release 应用商店系统）
- **b. 本项目的任务提出者、开发者、用户**
  - 任务提出者：项目发起人/产品经理
  - 开发者：GitGet 开发团队
  - 用户：GitHub 平台的所有开发者及普通用户
  - 运行环境：用户本地桌面操作系统（Windows/macOS/Linux）

### 1.3 定义

| 术语 | 定义 |
|------|------|
| SQLite | 嵌入式关系型数据库引擎，无需独立服务进程 |
| Microsoft.Data.Sqlite | .NET 官方 SQLite ADO.NET 提供程序 |
| Entity Framework Core | .NET 对象关系映射（ORM）框架，可选用于简化数据操作 |
| Cache | 本地缓存数据，用于减少 API 调用频率 |
| TTL | 生存时间（Time-To-Live），缓存数据的有效期 |

## 2 外部设计

### 2.1 标识符和状态

| 属性 | 值 |
|------|------|
| 数据库管理系统 | SQLite 3.x（通过 Microsoft.Data.Sqlite） |
| 数据库名称 | GitGet |
| 数据库文件名 | `gitget.db` |
| 数据库存放路径 | `{Environment.GetFolderPath(SpecialFolder.ApplicationData)}/GitGet/` |
| 使用方式 | 嵌入式（Embedded），应用程序直接访问本地文件 |
| 并发访问 | 单进程读写（SQLite 支持多线程读取，写入串行化） |
| 访问技术 | Microsoft.Data.Sqlite（首选）或 Entity Framework Core + SQLite Provider |

### 2.2 使用它的程序

| 程序模块 | 访问方式 | 说明 |
|----------|----------|------|
| CacheService | 读写 | 缓存仓库和 Release 数据 |
| DownloadService | 读写 | 持久化下载任务状态 |
| SettingsViewModel / SettingsService | 读写 | 用户配置存储 |
| RecommendService | 只读 | 读取缓存数据加速推荐 |

### 2.3 约定

- **表名命名规则**：全小写 + 下划线分隔（snake_case），如 `cache_repos`
- **字段命名规则**：全小写 + 下划线分隔（snake_case），如 `full_name`
- **主键字段**：统一命名为 `id`
- **时间字段**：统一使用 UTC 时间的 ISO 8601 字符串格式存储
- **JSON 字段**：存储复杂结构时使用 TEXT 类型，序列化为 JSON 字符串
- **禁用外键约束**：SQLite 默认外键约束关闭，应用层保证引用完整性

### 3.2 逻辑结构设计

#### 表 1：`cache_repos` - 仓库缓存表

| 列名 | 类型 | 长度 | 允许 NULL | 主键 | 说明 |
|------|------|------|-----------|------|------|
| id | INTEGER | — | NO | PK | 自增主键 |
| full_name | TEXT | — | NO | — | 仓库全名（如 "microsoft/vscode"），UNIQUE |
| data_json | TEXT | — | NO | — | 完整的 Repo JSON 数据 |
| cached_at | TEXT | — | NO | — | 缓存时间（ISO 8601, UTC） |
| expires_at | TEXT | — | NO | — | 过期时间（ISO 8601, UTC） |

**索引**：
- `idx_cache_repos_full_name` 唯一索引 on `full_name`
- `idx_cache_repos_expires_at` 普通索引 on `expires_at`（用于清理过期数据）

**SQL 建表语句**：
```sql
CREATE TABLE IF NOT EXISTS cache_repos (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    full_name   TEXT    NOT NULL UNIQUE,
    data_json   TEXT    NOT NULL,
    cached_at   TEXT    NOT NULL DEFAULT (datetime('now')),  -- UTC
    expires_at  TEXT    NOT NULL
);

CREATE INDEX IF NOT EXISTS idx_cache_repos_expires_at ON cache_repos(expires_at);
```

### 3.3 物理结构设计

#### 数据库文件

| 属性 | 值 |
|------|------|
| 数据库引擎 | SQLite 3.x |
| 数据库文件 | `gitget.db` |
| 文件路径（Windows） | `%APPDATA%/GitGet/gitget.db` |
| 文件路径（macOS） | `~/Library/Application Support/GitGet/gitget.db` |
| 文件路径（Linux） | `~/.local/share/GitGet/gitget.db` |
| 日志模式 | WAL（Write-Ahead Logging，提高并发读取性能） |
| 缓存大小 | 默认 2000 KB（约 2MB） |
| 页面大小 | 4096 字节 |
| 编码 | UTF-8 |

#### SQLite 运行时配置

```csharp
// C# 中初始化数据库时的推荐配置
var connectionString = new SqliteConnectionStringBuilder
{
    DataSource = dbFilePath,
    Mode = SqliteOpenMode.ReadWriteCreate,  // 自动创建数据库
    Cache = SqliteCacheMode.Shared,
    Pooling = true
}.ToString();

// 连接打开后执行 PRAGMA
using var connection = new SqliteConnection(connectionString);
await connection.OpenAsync();

// 启用 WAL 模式 - 提高并发读取性能
using var walCmd = connection.CreateCommand();
walCmd.CommandText = "PRAGMA journal_mode=WAL;";
await walCmd.ExecuteNonQueryAsync();

// 启用外键约束
using var fkCmd = connection.CreateCommand();
fkCmd.CommandText = "PRAGMA foreign_keys=ON;";
await fkCmd.ExecuteNonQueryAsync();

// 设置缓存大小（2000KB）
using var cacheCmd = connection.CreateCommand();
cacheCmd.CommandText = "PRAGMA cache_size=-2000;";
await cacheCmd.ExecuteNonQueryAsync();
```

#### C# 数据库初始化代码

```csharp
public class LocalDataStore : ILocalDataStore, IDisposable
{
    private readonly string _connectionString;
    private SqliteConnection? _connection;

    public LocalDataStore(string dbDirectory)
    {
        Directory.CreateDirectory(dbDirectory);
        var dbPath = Path.Combine(dbDirectory, "gitget.db");
        _connectionString = new SqliteConnectionStringBuilder
        {
            DataSource = dbPath,
            Mode = SqliteOpenMode.ReadWriteCreate,
            Cache = SqliteCacheMode.Shared,
            Pooling = true
        }.ToString();
    }

    public async Task InitializeAsync(CancellationToken ct = default)
    {
        _connection = new SqliteConnection(_connectionString);
        await _connection.OpenAsync(ct);

        // 配置
        await using var walCmd = new SqliteCommand("PRAGMA journal_mode=WAL;", _connection);
        await walCmd.ExecuteNonQueryAsync(ct);

        // 创建表
        await using var cmd = new SqliteCommand(@"
            CREATE TABLE IF NOT EXISTS cache_repos (
                id          INTEGER PRIMARY KEY AUTOINCREMENT,
                full_name   TEXT    NOT NULL UNIQUE,
                data_json   TEXT    NOT NULL,
                cached_at   TEXT    NOT NULL DEFAULT (datetime('now')),
                expires_at  TEXT    NOT NULL
            );
            CREATE INDEX IF NOT EXISTS idx_cache_repos_expires_at ON cache_repos(expires_at);
            
            CREATE TABLE IF NOT EXISTS cache_releases (
                id              INTEGER PRIMARY KEY AUTOINCREMENT,
                repo_full_name  TEXT    NOT NULL,
                data_json       TEXT    NOT NULL,
                cached_at       TEXT    NOT NULL DEFAULT (datetime('now')),
                expires_at      TEXT    NOT NULL
            );
            CREATE INDEX IF NOT EXISTS idx_cache_releases_repo ON cache_releases(repo_full_name);
            CREATE INDEX IF NOT EXISTS idx_cache_releases_expires_at ON cache_releases(expires_at);
            
            CREATE TABLE IF NOT EXISTS download_tasks (
                task_id         TEXT    PRIMARY KEY,
                repo_full_name  TEXT    NOT NULL,
                asset_url       TEXT    NOT NULL,
                file_name       TEXT    NOT NULL,
                file_path       TEXT    NOT NULL,
                total_bytes     INTEGER NOT NULL DEFAULT 0,
                received_bytes  INTEGER NOT NULL DEFAULT 0,
                status          TEXT    NOT NULL DEFAULT 'pending'
                                        CHECK(status IN ('pending','downloading','paused','completed','failed','cancelled')),
                progress        REAL    NOT NULL DEFAULT 0.0,
                error_message   TEXT,
                created_at      TEXT    NOT NULL DEFAULT (datetime('now')),
                updated_at      TEXT    NOT NULL DEFAULT (datetime('now'))
            );
            CREATE INDEX IF NOT EXISTS idx_download_tasks_status ON download_tasks(status);
            
            CREATE TABLE IF NOT EXISTS user_settings (
                key         TEXT    PRIMARY KEY,
                value       TEXT,
                updated_at  TEXT    NOT NULL DEFAULT (datetime('now'))
            );
        ", _connection);
        await cmd.ExecuteNonQueryAsync(ct);
    }
}
```

### 5.2 性能优化策略

| 策略 | 说明 |
|------|------|
| **WAL 模式** | 启用 SQLite WAL 日志模式，支持并发读取 |
| **索引优化** | 为查询频繁的字段建立索引（expires_at, status, repo_full_name） |
| **定期 VACUUM** | 每月执行一次 VACUUM 回收空间 |
| **预编译语句** | 使用 SqliteCommand.Prepare() 预编译高频执行的 SQL |
| **批量操作** | 多条 INSERT 使用事务包裹，减少磁盘 I/O |
| **缓存清理** | 每次启动时清理过期缓存 `DELETE FROM cache_repos WHERE expires_at < datetime('now')` |

## 6 安全设计

### 6.1 Token 安全存储

GitHub OAuth Token **不应存储在 SQLite 数据库中**。Token 使用 `System.Security.Cryptography` 进行 AES-256-GCM 加密后存储到独立文件 `token.enc`：

```csharp
public class SecureDataStore : ISecureDataStore
{
    private readonly string _filePath;
    private readonly byte[] _entropy;

    public SecureDataStore(string appDataDirectory)
    {
        _filePath = Path.Combine(appDataDirectory, "token.enc");
        _entropy = new byte[16];
        RandomNumberGenerator.Fill(_entropy);
    }

    public async Task SaveAsync(string key, object data, CancellationToken ct = default)
    {
        var json = JsonSerializer.Serialize(data);
        var plaintextBytes = Encoding.UTF8.GetBytes(json);

        // 使用 .NET Data Protection API (DPAPI) 在 Windows 上加密
        // 或使用 AES-GCM 手动加密
        var encryptedData = await EncryptAsync(plaintextBytes);
        await File.WriteAllBytesAsync(_filePath, encryptedData, ct);
    }

    public async Task<T?> LoadAsync<T>(string key, CancellationToken ct = default) where T : class
    {
        if (!File.Exists(_filePath))
            return null;

        var encryptedData = await File.ReadAllBytesAsync(_filePath, ct);
        var decryptedData = await DecryptAsync(encryptedData);
        var json = Encoding.UTF8.GetString(decryptedData);
        return JsonSerializer.Deserialize<T>(json);
    }

    public Task DeleteAsync(string key, CancellationToken ct = default)
    {
        if (File.Exists(_filePath))
            File.Delete(_filePath);
        return Task.CompletedTask;
    }
}
```

### 6.2 数据库文件安全

- 数据库文件存放在用户专属的 AppData 目录下，默认权限仅当前用户可访问
- SQLite 不支持内置加密，如需要可考虑使用 `sqlcipher` 替代
- 配置文件中的敏感信息（如代理密码）不应明文存储

## 7 实体关系（可选 - EF Core 方案）

如果使用 Entity Framework Core 访问 SQLite，对应的实体类：

```csharp
// GitGet.Core/Data/AppDbContext.cs

using Microsoft.EntityFrameworkCore;

namespace GitGet.Core.Data;

public class AppDbContext : DbContext
{
    public DbSet<CacheRepo> CacheRepos => Set<CacheRepo>();
    public DbSet<CacheRelease> CacheReleases => Set<CacheRelease>();
    public DbSet<DownloadTask> DownloadTasks => Set<DownloadTask>();
    public DbSet<UserSetting> UserSettings => Set<UserSetting>();

    public AppDbContext(DbContextOptions<AppDbContext> options) : base(options) { }

    protected override void OnModelCreating(ModelBuilder modelBuilder)
    {
        modelBuilder.Entity<CacheRepo>(entity =>
        {
            entity.ToTable("cache_repos");
            entity.HasKey(e => e.Id);
            entity.HasIndex(e => e.FullName).IsUnique();
            entity.HasIndex(e => e.ExpiresAt);
            entity.Property(e => e.DataJson).HasColumnName("data_json");
        });

        modelBuilder.Entity<CacheRelease>(entity =>
        {
            entity.ToTable("cache_releases");
            entity.HasKey(e => e.Id);
            entity.HasIndex(e => e.RepoFullName);
            entity.HasIndex(e => e.ExpiresAt);
        });

        modelBuilder.Entity<DownloadTask>(entity =>
        {
            entity.ToTable("download_tasks");
            entity.HasKey(e => e.TaskId);
            entity.HasIndex(e => e.Status);
            entity.Property(e => e.Status)
                  .HasMaxLength(20)
                  .HasConversion<string>();
        });

        modelBuilder.Entity<UserSetting>(entity =>
        {
            entity.ToTable("user_settings");
            entity.HasKey(e => e.Key);
        });
    }
}
```
