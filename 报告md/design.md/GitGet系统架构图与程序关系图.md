# GitGet 系统架构图与程序关系图

> 本文档提供 GitGet——GitHub Release 应用商店系统的完整系统架构图和程序关系图，展示核心模块之间的依赖关系和调用链路。
>
> **技术栈：C# + .NET 9 + Avalonia UI + SQLite**（全栈 C# 单一语言）

---

## 2. 调用关系与数据流图

### 2.1 用户操作数据流

```
用户操作        View层           ViewModel层          Service层             Data层        GitHub API
========       =======          ===========          ==========            =======       ==========
                                                                                          
[打开首页]  →  HomeView    →   HomeViewModel   →   RecommendService   →   CacheService
                        ↓                           ↓                     ↓
                    [显示加载中]                   [CheckCache]          [读取 SQLite]
                        ↓                           ↓                     ↓
                        ↓                        [缓存命中?] ←──────── [返回数据]
                        ↓                        ↓ 否    ↓ 是
                        ↓                   [GitHubApiClient]          [返回缓存]
                        ↓                        ↓
                        ↓                    [Search API] ──────────────────────────→ GET /search/repositories
                        ↓                        ↓                     
                        ↓                    [CacheService]   →   [写入 SQLite]
                        ↓                        ↓
                    [更新UI]  ←─────────── [更新 ObservableProperty]
                                                                                          
[搜索项目]  →  SearchView   →   SearchViewModel  →   GitHubApiClient    →   CacheService
                        ↓                           ↓                     ↓
                    [输入关键词]                  [CheckRateLimit]       [检查缓存]
                        ↓                           ↓                     ↓
                    [点击搜索]                   [Search Repos API] ─────────────GET /search/repositories
                        ↓                           ↓
                    [绑定结果]  ←─────────── [更新 SearchResults]
                                                                                          
[查看详情]  →  DetailView   →   DetailViewModel  →   GitHubApiClient    →   CacheService
                        ↓                           ↓                     ↓
                    [点击卡片]                   [GetRepoDetail]  ──────────────── GET /repos/{owner}/{repo}
                        ↓                           ↓
                        ↓                       [GetReleases]  ─────────────────── GET /repos/{owner}/{repo}/releases
                        ↓                           ↓
                        ↓                    [AssetMatcherService]  →   [匹配系统OS]
                        ↓                           ↓
                    [展示结果]  ←──────────── [更新 Repo/Releases]
                                                                                          
[下载文件]  →  DetailView   →   DetailViewModel  →   DownloadService    →   LocalDataStore   →   HTTP
                        ↓                           ↓                     ↓
                    [点击下载]                  [CreateTask]            [INSERT SQLite]
                        ↓                           ↓
                    [显示进度]  ←─────────── [StartTask (流式读取)] ─────────────────── GET {asset_url}
                        ↓                           ↓                     ↓
                    [进度条更新] ←─ [IProgress<DownloadProgress>]   [写入临时文件]  [读取流]
                        ↓                           ↓                     ↓
                    [完成通知]  ←──────────── [标记 Completed]          [重命名文件]
```

## 5. 关键设计模式

| 模式 | 使用位置 | 说明 |
|------|----------|------|
| **MVVM** | 整个 UI 层 | View（XAML）+ ViewModel（数据+命令）+ Model（数据模型） |
| **依赖注入（DI）** | 所有 Service 的创建和注入 | `Microsoft.Extensions.DependencyInjection` 容器统一管理生命周期 |
| **仓储模式（Repository）** | Data 层 | `ILocalDataStore` / `ISecureDataStore` 封装数据访问细节 |
| **策略模式** | IAssetMatcherService | 不同操作系统的匹配规则作为策略 |
| **观察者模式** | DownloadService | `event EventHandler<ProgressChanged>` / `IProgress<T>` 推送进度 |
| **工厂模式** | HttpClient 创建 | `IHttpClientFactory` 管理 HttpClient 生命周期 |
| **单例模式（Singleton）** | 全局服务 | CacheService, GitHubApiClient, TokenService 等注册为 Singleton |
| **异步编程模型** | 全部 IO 操作 | `async Task` / `CancellationToken` / `IProgress<T>` |
